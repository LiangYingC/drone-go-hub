# 產品定位：規劃輔助，非權威判定（含免責與同意記錄）

_最後更新：2026-06-19_

## Context

禁限航圖資為反解 CAA 前端、可能過期。若把資訊呈現為「權威判定」，使用者照做而違法或出事時，本站責任過重，因此定位為「規劃輔助」。此定位若只停在文案就沒有約束力，須落實為資料模型與 UI 的硬性不變量。V0 無登入、無後端、無 DB。

## Decision

產品定位為**規劃輔助**，非**權威判定**。落實為下列不變量：

- 用詞一律「參考／建議」，**不用**「合法／保證可飛」。
- 紅／黃／綠燈號為**建議性語意**，永遠配「參考」呈現。
- 資料模型**不得有讀起來像「判決」的欄位**（不要 `isLegal`／`canFly`），改用 advisory 類命名。
- 免責的**常駐入口不可移除**（固定位置、隨時可點開重看）；首次造訪**自動顯示、可關閉**（同意記錄存於瀏覽器 localStorage）。燈號卡片永遠保留「參考」字眼，並可一鍵連到完整免責。
- 「能不能飛非二元」：V0 紅／黃／綠是「**以一般娛樂飛行假設**」的簡化，此假設須在介面明示，並附**警語／連結至官方來源**供查證。
- 常駐標示**資料來源**與**圖資最後更新時間**。
- 記錄使用者對免責的**同意與時間**（做法見下）。

## 免責同意與身分驗證

**同意記錄（無登入如何做）**
- V0（無登入／DB）：首次造訪以**輕量一次性確認**取得同意，存於 **localStorage**：`{ disclaimerVersion, acknowledgedAt }`——在無帳號下也能保有「同意＋時間」的紀錄（以裝置為單位）。
- 免責文字／資料政策改版（`disclaimerVersion` 變動）時重新請求確認。
- 被動的常駐免責入口與此一次性確認**並存**。

**身分驗證模型（前瞻，非 V0 決策）**
- **匿名優先**：搜尋地址、查詢禁限航區、看天氣／Go-No-Go ＝ **無需登入**（低摩擦、即時價值）。
- **需登入**：個人化功能（收藏飛點、每日適宜度、社群回報），里程碑 2+。
- 登入後把 localStorage 的同意記錄**關聯／遷移到帳號**（DB 持久化）；匿名使用者以 localStorage 為準。
- 完整身分驗證決策待里程碑 2 的獨立 ADR（屆時標 `status: proposed`）。

## Considered Options

- **規劃輔助定位（採用）** — 用詞與 UI 皆向「參考」收斂，責任邊界清楚、與反解圖資的不確定性相符；也與「不追求權威以利變現」的策略一致。
- **權威判定定位（否決）** — 體驗上更乾脆（直接說「能不能飛」），但反解圖資可能過期，責任過重、法律風險高。

## Consequences

- **綠燈最危險**（最易被當成許可），故燈號卡仍標「參考」且須能一鍵點到完整免責。
- 首次的同意提示須**輕量、不打斷**「三秒看完」的即時價值，且**可關閉**，不要做成擋路的大彈窗。
- localStorage 同意是「**裝置級**非人級」：清快取／換裝置會遺失，V0 可接受。
- **適宜度評分（里程碑 1）**：不一定要有實質分數（可質化）；若採評分演算法，**演算法須公開透明**（公開公式與門檻，例如「風速 × 機重」的吃力門檻）。實際門檻／權重待研究（可能有官方評估方式），現階段僅記錄此原則、不定演算法。
- 社群層（v2）使用者貢獻內容另加免責：屬經驗分享、非官方許可。
