# 禁限航圖資的資料來源策略（法規驅動）

_最後更新：2026-06-27_

## Context

dronegohub 的核心是把台灣無人機的「禁限航圖資」（`regulatoryLayers`）呈現在地圖上供飛行前參考。要做這件事，得先回答：**法規上到底有哪些管制區、由誰公告、我們能從哪裡合法取得資料**。本 ADR 以現行法規為依據，定義圖層分類與資料來源策略。資料模型的 advisory 命名與「規劃輔助」定位見 [ADR 0008](0008-product-planning-aid-positioning.md)。

### 法規依據（已逐條查證，來源見文末）

- **《民用航空法》第 99-13 條第 1 項**：「禁航區、限航區及航空站或飛行場四周之一定距離範圍內，禁止從事遙控無人機飛航活動；航空站或飛行場四周之一定距離範圍由民航局公告之。」→ 禁航區、限航區、機場周邊為核心禁止區，機場那段「一定距離」由民航局公告。
- **第 99-13 條第 2 項**：「前項範圍外距地表高度不逾四百呎之區域，由直轄市、縣（市）政府……公告遙控無人機活動之區域、時間及其他管理事項。」→ **地方政府公告區**為獨立一類管制。
- **第 99-14 條**：操作限制——實際高度不得逾距地面或水面 400 呎、不得於日落後至日出前飛航、須在目視範圍內。→ 屬「操作限制」而非地理圖層，但為未來「適宜度評分／適飛時數」的法規輸入。
- **《遙控無人機管理規則》第 12 條**：「最大起飛重量一公斤以上且裝置導航設備……應具備主動限制或告警功能之圖資軟體系統，以防止……進入禁航區、限航區及航空站或飛行場四周之一定距離範圍」。→ **法規本身就要求一套禁限航圖資**；本產品等於該圖資的「飛行前查詢版」。
- **國家公園**：依《國家公園法》與各國家公園管理處公告，園區內操作無人機原則須事先申請許可——法源與民航法不同，獨立成一類。

### 開放資料現況（已查證）

- 核心的**禁航區／限航區**：民航局[遙控無人機管理資訊系統](https://drone.caa.gov.tw/)**只有互動查詢、無 API 或檔案下載**；政府資料開放平臺上「請開放禁限航 SHP／KMZ」的[建議案 136966](https://data.gov.tw/suggests/136966) 至今未獲釋出。→ **無官方開放下載資料**。
- **機場**：有[國內民航機場禁限建管制範圍圖資（35876）](https://data.gov.tw/dataset/35876)（KMZ、政府資料開放授權第 1 版）。⚠️ 但標題明寫「禁**限建**」（建築限高面），**語意不等於**無人機的機場禁飛緩衝（99-13 的「一定距離」）。
- **國家公園**：有[國家公園地理資訊圖層彙整（174421）](https://data.gov.tw/dataset/174421)（SHP／WMS、政府資料開放授權第 1 版、內政部國家公園署），含範圍圖。

## Decision

採**分層資料來源策略**，依「法規類型 × 開放資料可得性」決定每一層的來源；資料模型以法規分類為準。

### 1. 圖層分類（對齊法規 → `AdvisoryCategory`）

| 法規依據 | 圖層 | `AdvisoryCategory` | 來源策略 |
|---|---|---|---|
| 民航法 99-13 I | 禁航區 | `noFlyZone` | 官方公告數位化 |
| 民航法 99-13 I | 限航區（有條件） | `restrictedZone` | 官方公告數位化 |
| 民航法 99-13 I、規則 §12 | 機場周邊一定距離 | `airportBuffer` | 民航局公告幾何數位化；開放資料 KMZ 僅供「機場周邊參考」並標示「禁限建 ≠ 無人機緩衝」 |
| **民航法 99-13 II** | **地方政府公告區** | **新增 `localGovZone`** | 各縣市公告數位化（V0 可先列類別、資料留空） |
| 國家公園法＋管理處公告 | 國家公園 | `nationalPark` | **開放資料 ETL（174421 SHP／WMS）** |

### 2. 取得方式（依合法性與穩定性排序）

- **開放資料 ETL（優先）**：國家公園（174421）、機場參考（35876）→ 以 ETL 轉成靜態 GeoJSON，置於前端可載入的靜態檔，並以 GitHub Actions 定期更新。座標轉換：SHP 多為 TWD97（EPSG:3826）需轉 WGS84（EPSG:4326）；KMZ 依 KML 規格已是 WGS84，免轉。
- **官方公告數位化**：禁航區、限航區、地方公告區無開放檔，改以**官方公告**（民航局／縣市政府依行政程序法公告之公開資訊）人工數位化為 GeoJSON，每筆標**公告文號／來源 URL／公告日期**（對齊 `AdvisoryZoneProperties` 既有的 `source` / `sourceUrl` / `lastUpdated` 欄位）。
- **不採用反解 CAA 互動地圖端點**作為資料來源：該系統標「僅供參考」、未提供 API／下載授權、ToS 灰色且技術脆弱；與 ADR 0008 的責任邊界與本產品「不追求權威」一致。（最多作為人工核對時的私下對照，不寫入資料管線。）

### 3. 對資料模型的影響

- `AdvisoryCategory` **新增 `localGovZone`**（地方政府依 99-13 II 公告之區域）。
- `airportBuffer` 的 `advisoryNote` / `source` 必須標明「資料為機場禁限建管制範圍，非無人機飛航緩衝，僅供周邊辨識參考」。
- 操作限制（400 呎、日間、VLOS）以**設定／規則資料**保存，作為未來適宜度評分的法規輸入，不混入地理圖層。

## Considered Options

- **分層（開放資料 ＋ 公告數位化）＋ 不反解（採用）**
  - 授權明確（政府資料開放授權第 1 版可商用）、來源可標示、可離線、與「規劃輔助」定位一致。
  - 代價：禁限航需人工數位化與維護，資料可能不完整／有延遲——由「參考 ＋ 來源 ＋ 更新時間」承接（見 ADR 0008）。
- **反解 CAA 互動地圖端點（否決）**
  - 看似能取得最完整即時資料，同類網站（earthbook、boggy）實務上走此路。
  - 但無授權、ToS 灰色、端點未公開隨時可能變，且與「不追求權威」定位衝突、責任風險高。
- **等官方開放（否決）**
  - 最乾淨，但[建議案 136966](https://data.gov.tw/suggests/136966) 顯示官方短期無釋出計畫，等於無限期擱置產品核心。

## Consequences

- 本 ADR **取代** roadmap 中 M1「禁航圖資 pipeline（反解 CAA）」的方向，並**修訂 [ADR 0008](0008-product-planning-aid-positioning.md)** 對「圖資來源為反解 CAA」的早期假設（其資料不確定性論述仍成立：公告數位化同樣可能過期／不完整，故「規劃輔助」定位不變）。
- 新增 `localGovZone` 後，資料模型（`AdvisoryCategory`）與 `CONTEXT.md` 詞彙表須於實作 PR 補上對應條目。
- 上方「法規 → 來源策略」對應表即**未來修法時的檢查清單**：99-13 機場距離公告、地方公告變動、國家公園分區調整時，對應圖層與來源策略需回頭更新。
- V0 仍以示意假資料呈現（ADR 0008 已揭露）；本 ADR 確立的是 V0 之後接真資料的路線。機場層因語意差異，即使接上開放資料也須持續標示警語。

### 後續探索線索（待各自評估）

- **政府圖層補充來源**：國土測繪中心（NLSC）的公開地圖服務（`maps.nlsc.gov.tw`）可評估作為政府圖層與底圖的補充來源（關聯 [ADR 0004](0004-tech-stack-openfreemap-basemap.md)）；授權與圖層適用性待評估。
- **天氣資料來源候選**：天氣為獨立的 M1 ADR；除中央氣象署（CWA）外，**Open-Meteo**（免金鑰）為值得評估的候選，授權與資料適用性留待該 ADR 決定。
- **架構佐證**：同類產品常見作法為「先一次性 ingest 圖資、再由自家基礎設施（GIS server／後端 API）serve，而非執行期即時呼叫官方系統」，與本 ADR 的「ETL → 靜態 GeoJSON、自家 serve」一致；差異在本產品明確選擇合法的 ingestion 來源（開放資料＋公告數位化）。
