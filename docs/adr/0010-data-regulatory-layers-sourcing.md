# 禁限航圖資的資料來源策略（法規驅動）

_最後更新：2026-06-28_

## Context

dronegohub 的核心是把台灣無人機的「禁限航圖資」（`regulatoryLayers`）呈現在地圖上供飛行前參考。要做這件事，得先回答：**法規上到底有哪些管制區、由誰公告、我們能從哪裡合法取得資料**。本 ADR 以現行法規為依據，定義圖層分類與資料來源策略。資料模型的 advisory 命名與「規劃輔助」定位見 [ADR 0008](0008-product-planning-aid-positioning.md)。

### 法規依據（已逐條查證，來源見文末）

- **《民用航空法》第 99-13 條第 1 項**：「禁航區、限航區及航空站或飛行場四周之一定距離範圍內，禁止從事遙控無人機飛航活動；航空站或飛行場四周之一定距離範圍由民航局公告之。」→ 禁航區、限航區、機場周邊為核心禁止區，機場那段「一定距離」由民航局（交通部民用航空局，Civil Aviation Administration，縮寫 CAA）公告，並採民航法 §34「機場四周禁止施放有礙飛航安全物體」之範圍（見下）。
- **第 99-13 條第 2 項**：「前項範圍外距地表高度不逾四百呎之區域，由直轄市、縣（市）政府……公告遙控無人機活動之區域、時間及其他管理事項。」→ **地方政府公告區**為獨立一類管制。
- **第 99-14 條**：操作限制——實際高度不得逾距地面或水面 400 呎、不得於日落後至日出前飛航、須在目視範圍內。→ 屬「操作限制」而非地理圖層，但為未來「適宜度評分／適飛時數」的法規輸入。
- **《遙控無人機管理規則》第 12 條**：「最大起飛重量一公斤以上且裝置導航設備……應具備主動限制或告警功能之圖資軟體系統，以防止……進入禁航區、限航區及航空站或飛行場四周之一定距離範圍」。→ **法規本身就要求一套禁限航圖資**；本產品等於該圖資的「飛行前查詢版」。
- **國家公園**：依《國家公園法》與各國家公園管理處公告，園區內操作無人機原則須事先申請許可——法源與民航法不同，獨立成一類。

### 開放資料現況（已查證，2026-06）

民航局[遙控無人機管理資訊系統](https://drone.caa.gov.tw/)（下稱「CAA 互動地圖」）本身**只有互動查詢、無 API 或檔案下載**；但「請開放禁限航 SHP／KMZ」的[建議案 136966](https://data.gov.tw/suggests/136966) 已於 **2025-06 結案（有開放）**，民航局已將部分圖層上架開放資料：

- **限航區**：[臺北飛航情報區限航區範圍圖資（49021）](https://data.gov.tw/dataset/49021)（民航局、政府資料開放授權第 1 版）；可於[局網 a=243](https://www.caa.gov.tw/Article.aspx?a=243&lang=1) 下載 **KML／座標 Excel／PDF**。現行權威清單為 [eAIP `ENR 5.1`](https://ais.caa.gov.tw/eaip/AIRAC%20AIP%20AMDT%2002-26_2026_05_14/eAIP/RC-ENR%205.1-zh-TW.html) 之 **24 個 R 區**（含座標）。
- **機場／飛行場一定距離**：[機場四周禁止施放有礙飛航安全物體範圍圖資（45701）](https://data.gov.tw/dataset/45701)（依民航法 **§34**、民航局、開放授權第 1 版；KML／SHP／座標 Excel 等），**明文涵蓋「空拍機及遙控無人機」**；[99-13 說明](https://www.caa.gov.tw/Article.aspx?a=2429&lang=1) 證實 **99-13 機場「一定距離」即採 §34 範圍**（跑道端半徑 5 公里、左右各 35° 之弧；飛行場為 1 公里圓），故 45701 即此層對應圖資。
- **國家公園**：[國家公園地理資訊圖層彙整（174421）](https://data.gov.tw/dataset/174421)（內政部國家公園署、SHP／WMS、開放授權第 1 版），含範圍圖。
- **禁航區**：為法定類別，但現行 eAIP `ENR 5.1` 之禁航區（P）、危險區（D）均為 **NIL**（無常設區），**暫無地理資料**。
- **地方政府公告區**：**無集中開放資料**；民航局明示「屬各直轄市、縣（市）政府權責……請逕至各縣市政府公告網站查詢」，須逐縣市取得。

> ⚠️ **時效**：上述開放資料為「**不定期更新**」、部分下載檔可追溯至 2018 年，故現行清單以 **eAIP `ENR 5.1`** 為校驗基準；實際接資料時再多方交叉驗證（呈現方式屬實作／體驗設計階段）。
> ℹ️ 另有[國內民航機場禁限建管制範圍圖資（35876）](https://data.gov.tw/dataset/35876)，惟其為**建物高度管制（禁限建）**、與無人機飛航無關，**不採用**。

## Decision

採**分層資料來源策略**，依「法規類型 × 開放資料可得性」決定每一層的來源；資料模型以法規分類為準。

### 1. 圖層分類（對齊法規 → `AdvisoryCategory`）

| 法規依據 | 圖層 | `AdvisoryCategory` | 來源策略 |
|---|---|---|---|
| 民航法 99-13 I | 禁航區 | `noFlyZone` | 保留類別；現行 eAIP 為 NIL（暫無地理資料），待民航局公告 |
| 民航法 99-13 I | 限航區（有條件） | `restrictedZone` | **開放資料 ETL（49021）**，以 eAIP `ENR 5.1` 校驗現行清單 |
| 民航法 99-13 I（一定距離＝§34）、規則 §12 | 機場周邊一定距離 | `airportBuffer` | **開放資料 ETL（45701，§34、含無人機）** |
| **民航法 99-13 II** | **地方政府公告區** | **新增 `localGovZone`** | 各縣市公告數位化（無集中開放資料；V0 可先列類別、資料留空） |
| 國家公園法＋管理處公告 | 國家公園 | `nationalPark` | **開放資料 ETL（174421 SHP／WMS）** |

### 2. 取得方式（依合法性與穩定性排序）

- **開放資料 ETL（優先）**：限航區（49021）、機場一定距離（45701）、國家公園（174421）→ 以 ETL 轉成靜態 GeoJSON，置於前端可載入的靜態檔，並以 GitHub Actions 定期更新、對 **eAIP 現行清單校驗**。座標轉換：SHP 多為 TWD97（EPSG:3826）需轉 WGS84（EPSG:4326）；KML 依規格已是 WGS84；CSV／Excel 座標檔須先確認座標系統。
- **官方公告數位化**：**地方政府公告區**（`localGovZone`）無集中開放資料，須逐縣市以**官方公告**（縣市政府依行政程序法公告之公開資訊）人工數位化為 GeoJSON，每筆標**公告文號／來源 URL／公告日期**（對齊 `AdvisoryZoneProperties` 既有的 `source` / `sourceUrl` / `lastUpdated` 欄位）。禁航區目前無常設區（eAIP NIL），待民航局公告時比照辦理。
- **不採用反解 CAA 互動地圖端點**作為資料來源：禁限航幾何既已可由公開官方文件合法彙整取得（見上），即**無採此途之必要**；且本產品優先採用「授權明確、可標示來源」的資料，未經授權自第三方系統擷取資料與該原則及該系統服務條款不符（詳見 Considered Options）。該系統至多作為人工核對時的私下對照，不寫入資料管線。

### 3. 對資料模型的影響

- `AdvisoryCategory` **新增 `localGovZone`**（地方政府依 99-13 II 公告之區域）。
- `airportBuffer` 採 45701（民航法 §34「機場四周禁止施放有礙飛航安全物體」範圍，明文含無人機），其 `source` 標明法源為民航法 §34、資料集 45701；**不採用** 35876（禁限建／建物高度）。
- 操作限制（400 呎、日間、VLOS）以**設定／規則資料**保存，作為未來適宜度評分的法規輸入，不混入地理圖層。

## Considered Options

- **分層（開放資料 ＋ 公告數位化）＋ 不反解（採用）**
  - 授權明確（政府資料開放授權第 1 版可商用）、來源可標示、可離線、與「規劃輔助」定位一致。
  - 限航區、機場、國家公園走開放資料；僅地方公告區需逐縣市數位化。代價：開放資料為不定期更新、地方層需人工維護，資料可能不完整／有延遲——由「參考 ＋ 來源 ＋ 更新時間」承接（見 ADR 0008）。
- **反解 CAA 互動地圖端點（否決）**
  - 部分同類產品以此取得整合後的即時圖資，看似省去彙整工夫；惟對本產品而言，禁限航幾何已能由公開官方文件（eAIP `ENR 5.1`、民航局與縣市政府公告）合法取得，此途並非取得資料的必要手段。
  - 不採用的主因是**來源原則**：本產品優先採用授權明確、可標示來源的資料；未經授權自第三方系統擷取資料，與此原則及該系統服務條款不符。公開文件數位化的來源與授權明確、可追溯，與「規劃輔助」定位一致。
  - 該端點未公開、隨時可能改版或關閉，作為資料管線亦不穩定。
  - 採公開文件數位化的代價（完整度／時效較差）由本「分層」方案承接，並以「參考＋來源＋更新時間」揭露（見 ADR 0008）。
- **等官方開放（部分已實現）**
  - 限航區（49021）、機場（45701）已由民航局上架開放資料（[建議案 136966](https://data.gov.tw/suggests/136966) 於 2025-06 結案有開放），此部分已無須等待；惟**地方政府公告區**官方明示屬縣市權責、不集中提供，仍須逐縣市自行彙整。

## Consequences

- 本 ADR **取代** roadmap 中 M1「禁航圖資 pipeline（反解 CAA）」的方向，並**修訂 [ADR 0008](0008-product-planning-aid-positioning.md)** 對「圖資來源為反解 CAA」的早期假設（其資料不確定性論述仍成立：開放資料為不定期更新、公告數位化同樣可能過期／不完整，故「規劃輔助」定位不變）。
- 新增 `localGovZone` 後，資料模型（`AdvisoryCategory`）與 `CONTEXT.md` 詞彙表須於實作 PR 補上對應條目。
- 上方「法規 → 來源策略」對應表即**未來修法時的檢查清單**：99-13 機場距離公告、地方公告變動、國家公園分區調整時，對應圖層與來源策略需回頭更新。
- V0 仍以示意假資料呈現（ADR 0008 已揭露）；本 ADR 確立的是 V0 之後接真資料的路線。開放資料為不定期更新、部分檔較舊，接真資料時須以 eAIP 現行清單交叉校驗（呈現方式留待體驗設計階段）。

### 後續探索線索（待各自評估）

- **政府圖層補充來源**：國土測繪中心（NLSC）的公開地圖服務（`maps.nlsc.gov.tw`）可評估作為政府圖層與底圖的補充來源（關聯 [ADR 0004](0004-tech-stack-openfreemap-basemap.md)）；授權與圖層適用性待評估。
- **天氣資料來源候選**：天氣為獨立的 M1 ADR；除中央氣象署（CWA）外，**Open-Meteo**（免金鑰）為值得評估的候選，授權與資料適用性留待該 ADR 決定。
- **架構佐證**：同類產品常見作法為「先一次性 ingest 圖資、再由自家基礎設施（GIS server／後端 API）serve，而非執行期即時呼叫官方系統」，與本 ADR 的「ETL → 靜態 GeoJSON、自家 serve」一致；差異在本產品明確選擇合法的 ingestion 來源（開放資料＋公告數位化）。
